## Соответствие ТЗ по разделам

### §1 Цель задачи

| Требование | Статус | Реализация |
|------------|--------|------------|
| Универсальный модуль генерации XML | ⚠️ | `DocumentBuilderInterface`, `Documents/Upd/` — реализован только УПД |
| XML УПД по шаблону | ✅ | `UpdXmlWriter` (формат ФНС 5.03, образец Diadoc) |
| Задел на новые типы документов | ✅ | `Contract/`, `Documents/Upd/`, mapping |
| Задел на ЭДО | ✅ | `EdoGatewayInterface`, `NullEdoGateway`, `DocumentStatus` |

### §2 Область работ MVP

| Требование | Статус | Компонент |
|------------|--------|-----------|
| Получение данных из B24 | ✅ | `DataCollector` |
| Формирование XML УПД | ✅ | `UpdBuilder` → `UpdXmlWriter` |
| Проверка обязательных данных | ✅ | `ValidationMessages`, `UpdValidator`, preValidate |
| Сохранение XML | ✅ | `FileSaver`, `UF_UPD_FILE` |
| Запуск из интерфейса B24 | ✅ | `Event/Ui`, `ajax/generate.php`, `generate.js` |
| Логирование | ✅ | `Logger` → `b_xmldoc_log` |
| Отправка в ЭДО | ➖ | Не входит в MVP (по ТЗ) |

### §3 Источники данных

| Сущность | Статус | Примечание |
|----------|--------|------------|
| Сделка | ✅ | `TYPE_DEAL`, воронки 0/1 |
| Смарт-процесс «Счета» | ✅ | `TYPE_SMART_INVOICE`, entityTypeId в настройках |
| Компания | ✅ | `COMPANY_ID` → покупатель |
| Реквизиты компании | ✅ | `RequisiteTable`, DaData enrich |
| Банковские реквизиты | ✅ | `BankDetailTable` → mapper (`*_bank_*`) |
| Товары | ✅ | `ProductRowTable`, НДС, ОКЕИ |
| Пользовательские поля | ✅ | `UF_UPD_NUMBER`, `UF_UPD_FILE` |
| Карта полей B24 ↔ XML | ✅ | `config/mapping/upd.php` |

Банковские реквизиты собираются и попадают в mapped-данные. В эталоне `edo_6608_*.xml` узлов банка нет — в XML не выводятся (корректно для данного шаблона).

### §4 Функциональные требования

#### 4.1 Формирование XML

| Шаг | Статус |
|-----|--------|
| Данные из карточки сделки/СП | ✅ |
| Связанные сущности | ✅ |
| Проверка обязательных полей | ✅ |
| XML по шаблону | ✅ |
| Проверка структуры XML | ✅ well-formed; XSD — если указан путь |
| Сохранение | ✅ |

Pipeline:

```
GenerateService → DataCollector → UpdMapper → UpdValidator
  → UpdXmlWriter (WriterBuffer/DOM) → XmlValidator → XmlEncoder → FileSaver → DocumentRegistry
```

## Критерии приёмки MVP (чек-лист заказчика)

- [ ] Сделка с компанией, реквизитами, товарами
- [ ] Кнопка → JSON `success: true`
- [ ] Файл в `UF_UPD_FILE`, кодировка windows-1251
- [ ] Повторная генерация → v2 в `b_xmldoc_document`, оба file_id доступны
- [ ] Ключевые узлы XML совпадают с `edo_6608_*.xml`
- [ ] Пользователь без прав CRM получает отказ
- [ ] Робот CRM «Сформировать УПД (XML)» формирует документ
- [ ] История и журнал видны в админке

---

## Обновление на портале

1. Скопировать модуль → `/local/modules/vendor.xmldoc`
2. **Marketplace → Обновить** до **1.3.0**
3. Очистить кеш
4. Проверить: `/local/activities/xmldocgenerateupd/` появилась
5. CRM → Роботы → действие **«Сформировать УПД (XML)»**
6. Админка: **Настройки → XML УПД → История документов / Журнал**
